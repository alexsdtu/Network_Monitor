# Network Monitor

Десктопное приложение для периодического сканирования локальной сети и отслеживания доступности устройств. Показывает только те хосты, которые когда-либо отвечали на проверку, и наглядно отмечает устройства, которые перестали быть доступными.

## Какие задачи решает

| Проблема | Решение |
|----------|---------|
| Неизвестно, какие устройства сейчас в сети | Автоматическое сканирование выбранного диапазона IP и отображение отвечающих хостов |
| Нужно вручную пинговать адреса | Цикл проверки каждые **20 секунд** без участия пользователя |
| Список «засорён» всеми адресами подсети | В таблице только **активные** хосты и те, что **ранее были активны**, но пропали |
| Устройство отключилось незаметно | После **двух подряд** неудачных проверок хост помечается **красным** как потерянный |
| Непонятно, когда хост был в сети в последний раз | Для недоступных хостов фиксируется **время последней активности** |

Подходит для домашней или офисной LAN: контроль ПК, принтеров, камер, роутеров и другого оборудования с включённым ICMP (ping).

## Требования

- **ОС:** Windows (основная цель), также работает на Linux/macOS с установленной утилитой `ping`
- **Python:** 3.10 или новее
- **Зависимости:** только стандартная библиотека (`tkinter` входит в установщик Python для Windows)

## Установка и запуск

1. Установите [Python](https://www.python.org/downloads/) (при установке на Windows отметьте «Add Python to PATH»).
2. Откройте терминал в папке проекта:

```powershell
cd "d:\Network Monitor"
python network_monitor.py
```


## Как пользоваться

1. В поле **«Диапазон сети»** укажите подсеть или список адресов.
2. Нажмите **«Старт»** — начнётся первое сканирование, затем повтор каждые 20 секунд.
3. Следите за таблицей: зелёные строки — доступные хосты, красные — потерянные.
4. **«Стоп»** — остановка мониторинга.

### Форматы диапазона

| Формат | Пример |
|--------|--------|
| CIDR (подсеть) | `192.168.1.0/24` |
| Диапазон IP | `192.168.1.1-192.168.1.50` |
| Один адрес | `192.168.1.10` |

При сканировании более **1024** адресов программа запросит подтверждение.

## Как работает программа

```mermaid
flowchart LR
    A[Старт] --> B[Разбор диапазона IP]
    B --> C[Параллельный ping]
    C --> D[Обновление состояния хостов]
    D --> E[Таблица в GUI]
    E --> F{Мониторинг включён?}
    F -->|да| G[Пауза 20 с]
    G --> C
    F -->|нет| H[Стоп]
```

### Сканирование

- Для каждого IP выполняется **один ping** с таймаутом **1 с** (Windows: `ping -n 1 -w 1000`).
- Проверки идут **параллельно** (до 64 потоков), чтобы большая подсеть сканировалась быстрее.
- Ход сканирования отображается в строке прогресса.

### Состояния хоста

1. **Неизвестен** — адрес ещё ни разу не ответил; в таблице **не показывается**.
2. **Активен** (зелёный) — последняя проверка успешна; обновляется время последней активности.
3. **Потерян** (красный) — хост раньше отвечал, но **две проверки подряд** не прошли; в таблице остаётся с пометкой «Недоступен (потерян)» и временем **последнего успешного** ответа.

Если потерянный хост снова начинает отвечать, он возвращается в статус **«Активен»** (зелёный).

### Интерфейс

| Колонка | Содержание |
|---------|------------|
| IP-адрес | Проверяемый хост |
| Статус | «Активен» или «Недоступен (потерян)» |
| Последняя активность | Дата и время последнего успешного ping |

В строке состояния отображаются время последнего обновления, число активных и потерянных хостов, а также до следующего цикла.

## Настройка (в коде)

В начале файла `network_monitor.py`:

| Параметр | По умолчанию | Назначение |
|----------|--------------|------------|
| `SCAN_INTERVAL_SEC` | 20 | Интервал между циклами (секунды) |
| `FAILURES_BEFORE_LOST` | 2 | Сколько неудачных проверок подряд до статуса «потерян» |
| `PING_TIMEOUT_MS` | 1000 | Таймаут одного ping (мс) |
| `MAX_WORKERS` | 64 | Число параллельных проверок |

## Ограничения

- Проверка основана на **ICMP (ping)**. Устройства с отключённым ping на файрволе могут не отображаться, даже если они работают.
- Для некоторых сетевых операций может потребоваться запуск от имени администратора (зависит от политики ОС).
- Поддерживаются только **IPv4**.
- Очень большие подсети (например, `/16`) сканируются долго; для таких диапазонов лучше задавать узкий интервал IP.
